Docker에 대해서

2018.1.17

서론

Docker에는 이미 많은 사람들이 작성해서 올려놓은 이미지 파일들이 있다. Docker hub에서 원하는 키워드를 검색하여 손쉽게 다운로드 받아 러닝 시킬 수 있다. 하지만 아무리 많고 다양한 프로그램이 있다고 해도 내가 필요한 것을 직접 만드는 것만은 못할 것이다. 내가 필요한 환경을 구축할 수 있는 Docker 이미지를 만들고, 배포하면서 Docker에 대해 더욱 자세하게 알아볼 것이다.

Docker 이미지 만들기

이미지를 만들기 위해서는 우선 Dockerfile에 대해서 알아야 한다. Dockerfile의 설정방법에 대해서 알아보고 작성한 후 이미지를 생성하여 실행시킨다.

###Dockerfile이란

Dockerfile은 Docker 이미지 설정 파일이다. Dockerfile에 사용자가 설정한 순서대로 읽어서 이미지를 생성한다.

####Dockerfile 설정 명령어

  • FROM : 어떤 이미지를 기반으로 할지 설정한다. Docker 이미지는 기존에 만들어진 이미지를 기반으로 생성한다.

    <이미지 이름>:<태그>

  • MAINTAINER : 이미지를 생성한 사람의 정보를 설정한다.

  • RUN : 셸 스크립트 혹은 명령을 실행한다.

  • VOLUME : 호스트와 공유할 디렉터리 목록이다. docker run 명령에서 -v 옵션으로 설정할 수 있다.

  • CMD : 컨테이너가 시작되었을 때 실행할 실행 파일 또는 셸 스크립트이다.

  • WORKDIR : Docker 컨테이너 상에서 작업할 디렉토리를 설정할 수 있다.

  • EXPOSE : 호스트와 연결할 포트 번호이다.

  • ADD : Dockerfile과 같은 디렉토리에 있는 디렉토리를 추가하여 사용할 수 있다.

Dockerfile 작성 예시

FROM       ubuntu:16.04
MAINTAINER yljung@hibrain.net

ADD assets /assets
RUN /assets/setup.sh

EXPOSE 22
EXPOSE 1521
EXPOSE 8080

CMD /usr/sbin/startup.sh && /usr/sbin/sshd -D
  • FROM : 우분투 16.04 버전을 기반으로 이미지를 생성한다. 이미지를 실행하면 우분투 16.04가 설치된다.
  • MAINTAINER : 작성한 사용자의 정보이다.
  • ADD : Dockerfile과 같은 디렉토리에 있는 assets이란 디렉토리를 추가한다.
  • RUN : /assets/setip.sh 파일을 실행한다.

Build app

실행가능한 이미지로 생성하기 위해서는 docker build과정을 거쳐야 한다. 우선 build를 하기 전 작성한 Dockerfile은 사용자가 build할 대상이 되는 디렉토리에 넣는다. 기본적인 build 명령어 형태는 docker build 옵션 Dockerfile경로 이다.

Dockerfile build 옵션

docker build -t 저장소이름/이미지이름:태그 Dockerfile경로 : 저장소의 이미지, 이름, 태그를 설정

docker build -q Dockerfile경로 : RUN이 실행한 결과를 출력하지 않음

사용자가 작성한 Dockerfile의 설정에 따라 이미지가 생성되는 과정이 커맨드창에 뜰 것이다. 그리고 docker images ls로 생성된 이미지를 확인한다.

이미지 실행하기

생성한 Docker이미지는 docker run명령어로 실행시킬 수 있다.

이미지 실행 명령어는 docker run 옵션 이미지:태그 커맨드를 기본으로 한다.

-p : 포트 설정을 할 때 사용 호스트포트:컨테이너포트 로 포워딩이 가능

-d : detach 옵션으로 컨테이너를 현재 창에서 띄우지 않고 실행만 하도록 함

이미지는 docker rmi 이미지이름으로 삭제 할 수 있다.

컨테이너 실행하기

이미지를 사용하여 생성한 컨테이너를 실행시킬 수 있다.

docker exec 옵션 컨테이너아이디|이름 커맨트

실행된 컨테이너는 대신 docker stop 컨테이너아이디|이름으로 종료 시킬 수 있으며 docker rm 컨테이너아이디|이름으로 삭제할 수 있다.

Docker에 호스트 디렉토리 마운트하기

호스트의 디렉토리와 Docker 컨테이너의 디렉토리는 volume을 이용하여 공유 할 수 있다.

-v옵션으로 공유할 호스트 디렉토리와 Docker컨테이너에 표시 될 디렉토리 이름으로 공유 할 수 있다.

docker run -d -it -v "$(pwd)"/data:/app docker-ubuntu

컨테이너를 실행한 후 호스트의 디렉토리와 컨테이너의 디렉토리를 확인하면 같은 파일이 공유되는 것을 확인 할 수 있다.

docker exec -it 컨테이너ID/이름 /bin/bash

공유된 디렉토리는 파일 생성, 파일 삭제, 파일 수정이 모두 동일한 디렉토리처럼 공유되어 사용된다.
