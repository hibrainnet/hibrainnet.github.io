Docker에 대해서

2018.1.10

서론

Oracle환경을 실습하기 위해서는 먼저 Oracle 설치를 진행해야한다. 하지만 Oracle은 macOS나 Windows환경에서는 설치를 지원하지 않기 때문에 Ubuntu나 CentOS환경에 설치를 진행해야 한다. 실습용 개발 서버로 Ubuntu나 CentOS가 존재한다면 다행이지만, 존재하지 않는 경우도 있다. 이런 경우 보통 VMware나 VirtualBox를 이용하여 가상 OS환경을 만들어서 Oracle을 설치하였다. 가상 서버 환경에 Oracle을 설치하여 사용하면 동작은 하지만 무겁고 느리다는 단점이 발생한다. 이 때, Docker를 이용하면 기존의 VMware나 VirtualBox보다 가볍고 빠르게 가상 서버 환경과 연결하여 사용할 수 있다.


Docker란

Docker는 컨테이너 기반의 오픈소스 가상화 플랫폼이다. Docker는 소프트웨어를 컨테이너라는 표준화된 단위로 묶어주는데 해당 컨테이너 안에는 라이브러리, 시스템 도구, 코드 등 소프트웨어를 실행하는데 필요한 것들이 포함되어 있다. Docker를 사용하면 환경에 구애받지 않고 애플리케이션을 빠르게 배포 및 확장이 가능하다.


Docker의 가상화방식

*컨테이너와 가상화방식

컨테이너는 철판으로 만들어진 박스 형태의 통을 의미한다. 흔히 알고 있듯이 화물따위를 운송할 때 사용하는 통이다. 서버에서 컨테이너는 격리된 공간에서 프로세스가 동작하는 기술을 의미한다. 서버에서 컨테이너는 가상화 기술의 하나로, VMware나 VirtualBox가 이에 포함된다. VMware와 VirtualBox은 전가상화(Full Virtualization) 방식 기반인데 호스트OS 위에 게스트OS 전체를 가상화하여 사용한다. 이 방식은 여러 가지 OS를 가상화 할 수 있고 비교적 사용법이 간단하지만 무겁고 느리다는 단점을 가진다.


전가상화 방식 외에도 또다른 가상화 방식인 반가상화(Paravirtualization) 방식이 있다. 반가상화 방식은 게스트OS가 필요하지만 전체 OS를 가상화하는 방식은 아니다. 따라서 전가상화 방식보다는 빠르고 가볍게 사용할 수 있다.


*Docker의 가상화 방식

Docker는 위에서 설명한 반가상화 방식보다 좀 더 경량화된 방식이라고 할 수 있다. Docker에서 사용하는 가상화 방식은 전가상화나 반가상화 방식과 다르게 게스트OS를 설치하지 않는다. Docker이미지에 서버 운영을 위한 프로그램과 라이브러리만 격리하여 설치할 수 있으며, OS자원은 호스트와 공유하는 방식이다. 이 같은 방식을 사용하게 되면서 이미지 용량이 매우 줄어드는 장점이 있다. 또한 Docker는 하드웨어를 가상화하는 계층이 존재하지 않으므로 메모리 접근, 파일시스템, 네트워크 속도가 가상머신에 비해 매우 빠르다.


Docker를 이용함으로써 하나의 서버에서 여러 개의 컨테이너가 서로 영향을 미치치 않고 독립적으로 실행되면서 VMware와 다르게 매우 빠른 성능으로 사용이 가능해졌다. 실행중인 컨테이너에 접속하여 명령어가 입력가능하고 apt-get이나 yum으로 패키지 설치도 가능하다. 또한 호스트의 특정 포트와 연결하거나 호스트의 특정 디렉토리를 내부 디렉토리 인 것처럼 사용도 가능하다.


Docker의 컨테이너와 이미지

Docker에서 가장 중요한 개념은 컨테이너와 이미지이다.

이미지는 컨테이너 실행에 필요한 파일과 설정값 등을 포함하고 있는 것으로 변하지 않는다. 컨테이너는 이미지를 실행한 상태라고 볼 수 있으며 추가되거나 변하는 값은 컨테이너에 저장된다. 같은 이미지에서 여러 개의 컨테이너를 생성할 수 있고 컨테이너의 상태가 바뀌거나 컨테이너가 삭제되더라도 이미지는 변하지 않고 남아 있다.


Docker 이미지를 등록하고 직접 만들어 관리할 수 있도록 하기 위해서 Docker hub와 Docker Registry를 제공한다. Docker hub는 이미 만들어진 이미지를 다운받아 사용할 수 있도록 한 것이다. Docker Registry는 사용자가 이미지를 직접 만들어 관리할 수 있도록 한 것이다. 사용자의 컴퓨터에서 이미지를 관리하려면 용량도 매우 크기 때문에 불편함이 있었다. 하지만 Docker에서 제공해주는 위의 두 기능으로 용량 차지 없이 무료로 사이트에서 관리가 가능하다.


이미지는 컨테이너를 실행하기 위한 모든 정보를 가지고 있다. 때문에 새로운 서버 추가는 미리 만들어 놓은 이미지를 다운받고 컨테이너를 생성하는 간단한 방식으로 가능하다.



Docker를 이용하여 직접 Oracle을 설치하면 기존 방식의 단점을 보완할 뿐만 아니라, 더 좋은 기능들을 제공받으면서 사용할 수 있을 것이다.
