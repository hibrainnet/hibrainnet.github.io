하이브레인넷 맥 개발환경 설정

맥 개발환경 설정을 위해

  • 프로젝트 진행용 디렉토리 만들기
  • Homebrew 설치(nodenv, rbenv, pyenv, jenv)
  • python2.7과 3.6버전을 사용가능하도록 설치
    • python3.6의 환경이름 : python3.6
  • 개발에 필요한 툴
    • Jetbrain Toolbox(IntelliJ, WebStorm, DataGrip, Pycham Professional, Ruby Mine)
    • VisualStudio Code
    • Atom 에디터
  • Java 6, 7, 8, 9 설치
  • Ruby 2.3, 2.4 설치
  • Node.js 6.X, 7.X, 8.X LTS 버전 설치(가장 최신 버전)
  • Git Client
    • Tower2 (정품 라이센스 받기 전까지 Trial 버전 사용)
  • Postman 클라이언트 툴
  • 브라우저
    • Chrome
    • Canary Chrome
    • Firefox
    • Naver Whale
  • Docker for Mac
    • Docker에 OracleXE 설치

순서로 설치를 진행한다.

프로젝트 진행용 디렉토리 만들기

프로젝트 진행용 디렉토리는 루트 디렉토리 밑에 /Projects로 생성한다.

cd /

sudo mkdir Projects

해당 디렉토리는 사용자 접근 권한을 부여하여 사용한다.

ls -l 권한 확인이 가능하다.

sudo chown -R 사용자이름 Projects/

/Projects밑에 /Repositories/Hibrainnet 디렉토리를 생성하여 현재 개발 중인 프로젝트의 로컬 저장소로 사용한다.

권한을 준 후에는 사용자에게 권한을 주었기 때문에 sudo명령어를 붙일 필요가 없다.

/Projects 안에 들어가서 /Repositories/Hibrainnet 디렉토리를 생성한다.

mkdir Repositories
cd Repositories
mkdir Hibrainnet

Homebrew 설치

Homebrew는 macOS용 패키지 관리자이다. Apple에서 제공하지 않는 패키지들을 검색, 설치, 업데이트 등을 할 수 있다.

Homebrew홈페이지에서 제공하는 명령어를 이용하여 설치를 진행한다.

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

  • nodenv, rbenv, pyenv, jenv 설치
    • 위의 네가지 패키지는 nodejs, ruby, python, java의 버전을 관리하기위해 설치하는 패키지이다.
    • brew install 패키지이름을 이용하여 각 패키지를 설치한다. (ex. brew install nodenv)

Pyton 설치

Python 2.7과 3.6 두 가지 버전으로 설치하여 사용한다.

설치된 Python은 virtualenv으로 가상환경에서 여러 버전을 사용하기 위해 아래의 프로그램을 설치한다.

  • pip
  • virtualenv

Python설치 시 virtualenv를 이용하여 설치한다.

sudo easy_install pip 명령어를 이용하여 pip을 먼저 설치한다.

virtualenv는 pip의 버전이 1.3이상이어야 하므로 pip -V를 이용하여 pip의 버전을 확인한다.

설치한 pip을 이용하여 virtualenv는 설치한다.

pip install virtualenv

penv를 이용하여 각 Python 버전의 패키지를 설치한다.

pyenv install 2.7

pyenv install 3.6.0

pyenv versions

원하는 디렉토리에 아래의 명령어로 설치한다.

설치된 Python2.7, 3.6버전 별로 아래의 이름으로 가상 공간을 만들어준다.

  • python2.7
  • python3.6

virtualenv --python=/Users/yljung/.pyenv/versions/2.7/bin/python2.7 python2.7

만들어진 가상공간 디렉토리 안에서 python을 실행한다.

source bin/activate

실행을 종료한다.

deactivate

설치한 Python은 원하는 버전을 사용하여 가상공간을 만들어 사용가능하다.

개발에 필요한 툴

Jetbrain Toolbox

IntelliJ, WebStorm, DataGrip, Pycham Professional, Ruby Mine

Jetbrain의 Toolbox는 라이센스를 구매하여야 하는 유료 툴이다. 트라이얼 버전으로 30일 동안 무료로 사용가능하다.

intellij의 경우 무료로 제공하는 community툴이 있지만 상업적용도의 개발은 할 수 없다.

학생인 경우 학생 메일을 이용하여 유료 라이센스를 무료로 받을 수 있는 혜택이 있다.

VisualStudio Code

VisualStudio Code 홈페이지에서 파일을 다운받아 설치를 진행한다.

Atom 에디터

Atom 홈페이지에서 파일을 다운받아 설치를 진행한다.

JAVA 설치

JAVA는 6, 7, 8, 9버전을 설치하여 사용한다.

jenv를 이용하여 버전을 관리할 수 있도록 한다.

brew cask install java을 이용하여 자바를 설치한다.

java 뒤에 버전을 명시하지 않는 경우는 자동으로 가장 최신 버전이 설치된다.

JAVA 6, 7, 8, 9를 설치해야 하므로 각 버전 별로 설치를 진행한다.

brew cask install java9

brew cask install java8

brew cask install java7

brew cask install java6

설치를 하던 중 java7의 경우 cask에 존재하지 않는 이름이라는 에러가 난다. 그래서 java7은 직접 사이트에서 다운로드 받아 설치를 진행하였다.

설치를 완료한 JAVA를 jenv에 추가해준다.

jenv에 추가하기전 jenv의 JAVA경로를 초기화해준다.

if which jenv > /dev/null; then eval"$(jenv init -)"; fi

설치한 JAVA Home경로를 입력하여 jevn에 추가해준다.

jenv add /Library/Java/JavaVirtualMachines/jdk7/Contents/Home/

***자바 경로를 확인 : **/usr/libexec/java_home -v 1.9

시스템 전체에서 사용한 자바 버전을 지정할 때 아래와 같이 사용한다.

jenv global 버전

특정 경로에서 사용할 자바 버전을 지정할 때 아래와 같이 사용한다.

cd <my project> jenv local 버전

Ruby 설치

Ruby는 2.3과 2.4버전을 설치한다.

rbenv install -list를 이용하여 설치가능한 버전을 확인한다.

설치하고자 하는 버전으로 Ruby를 설치한다.

rbenv install 2.3.0

rbenv install 2.4.0

Node.js 설치

Node.js는 6.X, 8.X LTS 버전을 설치한다.

nodenv install -l를 이용하여 설치가능한 버전을 확인한다.

LTS버전 확인은 Node.js 홈페이지에서 가능하다.

nodenv install 8.9.4

nodenv install 6.12.3

Git Client

Tower2를 설치한다. 정품 라이센스를 받기 전까지 Trial 버전을 사용한다.

Tower 홈페이지에서 파일을 다운받아 설치를 진행한다.

Postman 클라이언트 툴

Postman은 API 개발, 테스트, 팀원들간 공유를 할 수 있게 해주는 플랫폼이다.

Postman 홈페이지에서 파일을 다운받아 설치를 진행한다.
